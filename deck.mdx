# The What

- Typescript is just javascript + (optional) typesystem which compiles to regular javascript
  - developed by Microsoft
  - included in VSCode

---

# Why would you use typescript?

- VSCode autocompletion (function params, react props)
- Way better refactoring
- Maybe even prevents bugs ðŸ¤ª

---

Typescript adds a **static** type system

---

- All javascript is valid typescript
- Your code will always "compile"
- Compiling == removing the type information (example)

---

# Type system

- Think of the typechecker and compiler as separate
- The type system is gradual (types are optional)
- Treat the errors as **"just"** suggestions !!!
- The types can be "separate" from the code [https://definitelytyped.org/](https://definitelytyped.org/)

---

# Fun fact

TS type system is turing complete [https://github.com/Microsoft/TypeScript/issues/14833](https://github.com/Microsoft/TypeScript/issues/14833)

# How

- Opt in for a single file: <code style={{color: "#000f00" }}>// @ts-check</code>
- Start a new react project: <code style={{color: "#000f00" }}>npx create-react-app my-app --typescript</code>
- Add to an existing project using babel:
  - <code style={{color: "#000f00" }}>npm i -D @babel/preset-typescript</code>
  - create a <code style={{color: "#000f00" }}>tsconfig.json</code> and add a <code style={{color: "#000f00" }}>noEmit: true</code> option
